Data Types

Beginners to JavaScript, welcome! In this first article of the many that will follow of the JavaScript for Beginners Hub series, we will learn about the 8 basic Data types. We will get confused too but no worries, the hand-full of amazing videos and articles of the top educators are coming to our aid. Ready to dive in?

Numbers

In JavaScript, numbers are what they are in maths: regular numbers of any kind, from Whole numbers (0,1,2,...) to Real Numbers (0.4, -12.7 etc). Number is also considered the NaN value (not a number) that results from an operation of a number and a non-number e.g. "Hello"/2 = NaN.

For a better insight, watch FreeCodeCamp's video.

BigInt

Integers (25, 8, 9640) are numbers, however, integers of arbitrary length belong to the BigInt type. In other words, all values larger than (2**53-1) or less than -(2**53-1) are BigIntegers. The "n" in the end of a value is also indicating a BigInt e.g x = 65412n.

Time to study and practise on this awesome MDN article.

Strings

A string is anything in quotes, either double or single (" ", ' '), for example "Hello", "We're very happy". Numbers in quotes are also strings e.g. "52".

However, strings are a wide JS topic. This W3schools article will answer all your questions.

Undefined

An undefined value is simply a value that has not been assigned unintentionally. For example if we declare a variable let = age; and then we run the alert function (alert(age);), the outcome will be "undefined" as we forgot to assign a value to variable "age".

Null

A null value on the other hand represents nothing, an empty and unknown value, for example let Year = null.

Ok, null means an empty value, so what? Null and undefined might be confusing at start but here comes a FreeCodeCamp video to save the day and make the difference more understandable.

Boolean

Booleans are two; true and false, always written with lower case letters.

For example let price = 5 > 1 alert(price); will give out true since 5 is indeed greater than 1.

This super article from W3Schools not only will introduce booleans but also comparisons!

Objects

The object type is special and very important in JavaScript. Objects are used to store collections of data.

For example, let person = { firstName: "John", age: 26 };. Person is an object, firstName and age are properties and the curly brackets{} define an object.

This super Mosh video will give a more detailed overview of objects.

typeof Operator

This operator is used in our console to find out the data type of value. For example if in our console we write 'typeof 5' it will gives us "number" because 5 is a number. If we write 'typeof true', we will get "boolean". So in case you forgot the data type of NaN, you can use typeof and voila!
